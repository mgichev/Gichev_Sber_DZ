Структура программы:
Класс TerminalUI является классом, через который взаимодействует пользователь
Для простоты решения задачи будем задавать команды для этого класса через код

Этот класс использует класс TerminalImpl для запуска необходимых задач.
Класс TerminalImpl вызывает необходимые методы у компонентов программы:
    1) У класса PinValidator - обработка попытки войти в аккаунт
    2) У класса TerminalServer - операции со счетом в случае успешной авторизации

Класс PinValidator запрашивает у пользователя данные для входа и ищет совпадения в своей БД пользователей
Для простоты часть UI и обработки ошибок была добавлена в этот класс
Генерирует ошибки в случае неправильно вводимых данных (PinWrongInputException)
Либо слишком частых попыток авторизации  (AccountIsLockedException)
В случае неудачной авторизации (после 3 попыток) выбрасывается исключение WrongAuthorizationException
Блокирует аккаунт на 10 секунд в случае неуданочной авторизации

TerminalServer совершает операции со счетом
Если были введены неправильные данные либо на счету недостаточно средств, выбрасывается исключение WrongSumException

TerminalUI обрабатывает ошибки, выводит дополнительную информацию
и вызывает статический метод showException класса ExceptionsUI для вывода сообщений в удобном виде на экран
